> **今日看点**：今日信号很杂，但主线清晰：AI 正在从“能对话”向“会动手、能干活”冲刺。browser-use 和 ASPIRE 分别从数字界面和物理机器人两个方向验证了 agent 自主编程、安装技能的可能性，这让“AI 员工”不再只是聊天框里的幻觉。感知层也在经历一次静默升级——PerceptionRubrics 和 PixelEyes 都指向同一个方向：多模态模型在感知与推理之间出现了一个 8% 的认知裂缝，谁先弥合这个裂缝，谁就能突破基准饱和。而 LobeChat 和 LiteLLM 一天内刷出十几个小版本号，说明开源生态正在用高频迭代对抗模型能力的碎片化——工具层的竞争已经从“能用”变成了“修得快”。

### PerceptionRubrics 用双流评分标准突破多模态基准饱和，揭示开源闭源 8% 感知差距

PerceptionRubrics 提出基于评分标准的评估框架，用 1038 张高密度图像和 12000 余条实例标准，将评估从整体语义匹配转向原子级审查。框架通过循环同行评审生成黄金标注，并提炼为 Must-Right（必须正确）与 Easy-Wrong（易错细节）双流标准，配合门控评分机制，在关键事实错误时触发硬性惩罚。测试发现：模型常能正确校验碎片元素，却无法满足严格合取约束，暴露密集场景下的脆弱性；开源与闭源模型之间存在 8% 的感知能力差距；

#PerceptionRubrics #多模态评估 #感知可靠性
依据：[PerceptionRubrics 使用 1038 张信息密集图像和超过 12000 条实例特定评分标准](https://huggingface.co/papers/2606.28322)；[框架采用 Must-Right 和 Easy-Wrong 双流评分标准，并实现门控评分机制，关键事实错误导致二元惩罚](https://huggingface.co/papers/2606.28322)；[评估揭示开源和闭源模型之间存在 8% 的感知能力差距，且门控指标与人类对齐度显著优于传统基准](https://huggingface.co/papers/2606.28322)

### MemSyco-Bench：首个检测 Agent 记忆诱导的谄媚倾向基准

LLM Agent 依赖记忆实现长期协作，但记忆也可能引发“谄媚”问题，即 Agent 为迎合用户偏好而牺牲事实准确性。现有记忆基准仅评估存储与更新，忽略了记忆对下游推理的负面影响。MemSyco-Bench 填补这一空白，通过五项任务测试 Agent 能否拒绝将记忆作为事实、尊重记忆适用范围、化解记忆与客观证据冲突、追踪记忆更新，以及利用记忆实现个性化。对开发者意味着，部署 Agent 时需警惕记忆导致的“伪对齐”，并应引入类似的反谄媚评估机制。

#AgentMemory #Sycophancy #LLMBenchmark
依据：[MemSyco-Bench 主要衡量记忆引发 Agent 系统谄媚的五项任务，包括拒绝记忆作为事实证据、尊重适用范围、解决记忆与客观证据冲突等。](https://huggingface.co/papers/2607.01071)
