## 论文

### 从零胜到6/10胜：AgenticSTS 用有界记忆为长时域 LLM 代理打开新局面

AgenticSTS 在《Slay the Spire 2》中构建了一个有界记忆测试台，要求 LLM 代理在数百次战术与战略决策中不依赖原始对话历史，而是通过类型化检索组装新鲜提示，使提示长度不随运行长度增长。在最低难度下，前沿 LLM 公开基准的胜率为零，而人类胜率为 16%。固定消融实验显示，加入触发式战略技能层后，无存储基线胜率从 3/10 提升至 6/10。

#LLM代理 #记忆机制 #基准测试
依据：[在最低难度下，前沿 LLM 的公开基准报告零胜率，开发者报告的人类胜率为 16%](https://huggingface.co/papers/2607.02255)；[无存储基线胜率 3/10，加入触发式战略技能层后胜率提升至 6/10](https://huggingface.co/papers/2607.02255)；[发布了包含 298 条完整轨迹、条件标签、冻结记忆/技能快照、提示记录和分析脚本的可复现测试平台](https://huggingface.co/papers/2607.02255)

### AutoMem让LLM学会记忆管理，32B开源模型性能提升2-4倍，追平Claude、Gemini

AutoMem将记忆管理视为可训练认知技能，通过双循环框架自动优化记忆结构与模型决策能力。第一个循环中，强模型复盘完整轨迹，迭代修正记忆相关提示、文件模式；第二个循环则从大量经验中提取正确记忆决策，直接训练模型。在Crafter、MiniHack、NetHack三款长程游戏中，仅优化记忆机制，不调整任务行为，就将32B开源模型性能提升2-4倍，使其与Claude Opus 4.5、Gemini 3.1 Pro Thinking等前沿系统竞争。

#记忆管理 #长程任务 #LLM训练
依据：[AutoMem通过双循环自动优化记忆结构和模型熟练度，在三个长程游戏中仅优化记忆就将32B开源模型性能提升2-4倍，达到与闭源前沿系统竞争的水平。](https://huggingface.co/papers/2607.01224)
来源 [hf-papers](https://huggingface.co/papers/2607.01224) · 93 分

### LiteLLM v1.92.0-dev.2 发布：Docker 镜像签名验证与多项新特性

LiteLLM v1.92.0-dev.2 已发布，所有 Docker 镜像均使用 cosign 进行签名，用户可通过固定 commit hash 或 release tag 验证签名。该版本同时引入了 MCP 工具搜索、模型预算回退、Google Distributed Cloud Gemini 提供商等多项功能。对从业者而言，镜像签名验证增强了供应链安全性，而新特性则提升了代理工作流的灵活性和成本控制能力。

#LiteLLM #Docker镜像签名 #MCP
依据：[所有 LiteLLM Docker 镜像均使用 cosign 进行签名，每个版本都使用同一密钥签名。](https://github.com/BerriAI/litellm/releases/tag/v1.92.0-dev.2)；[提供两种验证方式：使用固定的 commit hash 或 release tag。](https://github.com/BerriAI/litellm/releases/tag/v1.92.0-dev.2)
来源 [litellm-gh](https://github.com/BerriAI/litellm/releases/tag/v1.92.0-dev.2) · 97 分

### LiteLLM v1.90.1 发布，新增 Docker 镜像签名验证

LiteLLM 发布 v1.90.1，所有 Docker 镜像均使用 cosign 签名，防止篡改。提供两种验证方式：推荐使用不可变的 commit hash 验证，也可使用 release tag 验证。对从业者意味着可以更安全地部署可信的 LiteLLM 容器，尤其适合生产环境中的供应链安全审计。
