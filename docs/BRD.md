## 一、项目背景与初衷 (Project Background)

> **底层逻辑：** 明确"现状"与"理想"之间的鸿沟。如果没有这个痛点，项目就没有存在的必要。

| 模块 | 内容描述 |
| --- | --- |
| **项目名称** | AI News Daily (每日 AI 新闻摘要) |
| **项目背景** | 2024-2026 年生成式 AI 进入工业级落地阶段，论文 / 模型 / 工具的发布频率远超人工跟踪的上限。中文技术媒体普遍存在「二手搬运 + 延迟 24-72h + 标题党」三大问题，而英文一手源又分散在 50+ 个 RSS / API 上。 |
| **痛点分析** | 1. **信噪比低**：中文公众号 80% 以上是搬运稿，且常常漏掉真正的关键论文（如 HF Papers 高赞）<br/>2. **时效差**：从英文一手源 → 翻译 → 中文渠道平均滞后 1-3 天，错过讨论窗口<br/>3. **来源碎片**：研究者要同时盯 HF、arXiv、OpenAI Blog、DeepMind Blog、GitHub Trending、X 等 10+ 渠道，每天耗费 1-2 小时 |
| **迫切性** | 1. AI 节奏从「周更」进入「日更」（GPT-4.5、Sonnet 4.6、Opus 4.7 半年内三次迭代）<br/>2. RSSHub 公共实例 2026-05 起全部 403，原有依赖该工具的中文资讯渠道大面积失能，**留下了短期空窗期**<br/>3. Claude / GPT 已具备稳定的标题翻译 + 摘要生成能力，技术成本从 2024 年起下降 10× 以上 |

## 二、市场分析与定位 (Market Analysis)

### 2.1 市场规模 (Market Size)

- **TAM**：中文技术从业者中关注 AI 的人群约 200-400 万（参考 InfoQ、CSDN、机器之心订阅量级），每年信息工具市场规模数亿元。
- **SAM**：愿意阅读英文一手摘要、追求高质量信息的研究者 / 工程师 / 高级 PM，估计 5-20 万人。
- **SOM（初期可触达）**：作者自身的同行圈子 + Notion 公开页面 SEO 长尾，预期 1k-5k MAU。

### 2.2 竞品分析 (Competitor Analysis)

**直接竞品：**

| 竞品 | 模式 | 优势 | 劣势 |
| --- | --- | --- | --- |
| 量子位 / 机器之心 | 编辑团队 + 公众号 | 受众大、内容深 | 严重二手搬运、日更慢、AI 选题偏新闻 |

**间接竞品：**

**差异化定位：**

- **一手 + 中文 + 结构化** 三者交集是空白区
- **可归档到 Notion**，形成个人 / 团队知识库，而非阅后即焚

## 三、解决方案与产品概览 (The Solution)

- **产品定位**：为 **AI 研究者与工程师** 提供 **每日 7 条经评分筛选的英文一手 AI 资讯中文摘要** 的 **自动化 Notion 日报系统**。
- **核心功能亮点**：
  1. 50+ 一手源并行采集（HF Papers/Models 为主源，RSS / GitHub 为辅源）
  2. 0-100 分质量评分 + 类型配额（论文 1、模型 1、工具 1-2、社区 1、官方 1-2、新闻 0-1）
  3. 同事件多源去重聚类，杜绝重复
  4. 英文标题 → 中文翻译（技术术语保留英文）+ < 120 字符摘要
  5. 一键发布到 Notion，自带 Quality / Tags / Date 等属性，可检索
- **产品路线图 (Roadmap)：**
  - **Phase 1（MVP，已完成 P0/P1）**：核心采集 → 处理 → Notion 发布链路打通
  - **Phase 2（短期 P2）**：X/Twitter 热词 + 每日 Cron 自动跑
  - **Phase 3（中期 P3）**：Web Dashboard，可视化配置来源 / 阈值 / 历史回看
  - **Phase 4（长期）**：开源 + 多用户多 Notion 库版本 → 潜在 SaaS

## 四、商业模式与盈利预测 (Business Model)

> 本项目当前定位为**个人研究工具 + 公开 Notion 站点**，不以直接盈利为首要目标，但保留商业化选项。

1. **盈利模式（按可行性排序）：**
   - **L0 — 当前**：免费 + 个人公开 Notion 页面，沉淀作者品牌
   - **L1 — 半年内**：开源核心代码 → 通过 GitHub Sponsors / 知识星球收赞助
   - **L2 — 一年内**：托管版 SaaS，用户输入自己的 Notion token + 偏好领域，5-20 元/月
   - **L3 — 长期**：企业版（团队共享 Notion 库 + 自定义来源 + 私有部署），200-2000 元/月

2. **核心指标 (North Star Metric)**：
   - **早期**：每日 Notion 页面发布成功率 ≥ 95% + 每日 Notion 阅读用户 (DAU)
   - **中期**：订阅 / 收藏的研究者数量
   - **长期**：付费用户数 × ARPU

3. **收益预测（粗估）：**
   - **保守**：保持个人项目，年支出 = 年收入 ≈ 0（API 费用与赞助抵消）
   - **乐观**：一年内 500 付费用户 × 10 元/月 = 6 万元/年；三年内企业客户 20 家 × 5000 元/年 = 10 万元/年

## 六、风险评估与对策 (Risk Management)

| 风险类别 | 风险描述 | 对策 |
| --- | --- | --- |
| **技术风险** | 1. 单一源失效导致采集为空<br/>2. LLM 翻译漂移 / 幻觉<br/>3. Notion API 限速或 schema 变更 | 1. 50+ 源冗余 + 自动 failover + 状态注册表<br/>2. 仅翻译标题、术语保留英文，摘要走规则模板 + LLM 后修订<br/>3. 失败回退到本地 Markdown，PRD 4.3 已定义 |
| **市场风险** | 1. 中文 AI 资讯渠道格局变化（大厂入场）<br/>2. 用户习惯仍偏向公众号而非 Notion | 1. 早期专注研究者圈层，不与大流量赛道正面竞争<br/>2. 输出多通道：未来加 RSS / Newsletter 输出 |
| **合规风险** | 1. 转载英文标题 / 摘要的版权问题<br/>2. 用户 Notion token 等凭证存储合规 | 1. 仅做评分 + 中文摘要（属合理使用 fair use），原文链接保留<br/>2. 个人版仅本地存储，SaaS 阶段加密 + 不持久化原始 token |
| **运营风险** | 作者断更 / 项目失维 | 全自动 cron + 异常告警 + 异常路径已在 PRD 4.3 定义；开源后社区可接手 |

## 七、结论与建议

- **项目价值总结**：在 RSSHub 失效 + LLM 成本崩塌 + AI 节奏加速的三重窗口期，AI News Daily 用极低运维成本，填补「中文一手 AI 日报」的空白，先服务作者自身研究，再以零边际成本对外公开。已完成 P0/P1（核心采集 + 评分 + Notion 发布），下一步是补齐 P2 自动化 + 沉淀公开品牌。
